## 2. 収益シミュレーション（現実的な見通し）

### AdSense収益の計算式

```
月間収益 = 月間PV ÷ 1,000 × RPM
```

### テック系ブログの平均値

| 指標 | 数値 |
|------|------|
| CPC（クリック単価） | 25〜50円 |
| CTR（クリック率） | 0.5〜1.5% |
| RPM（1,000PVあたり収益） | 200〜500円 |

### 月3万円達成に必要なPV

| シナリオ | RPM | 必要月間PV |
|----------|-----|-----------|
| 楽観 | 400円 | 75,000 PV |
| 現実的 | 250円 | 120,000 PV |
| 保守的 | 150円 | 200,000 PV |

**目標: 月間10〜12万PV**

### 達成までのロードマップ

| 期間 | 累計記事数 | 月間PV予測 | 月間収益予測 |
|------|-----------|-----------|-------------|
| 1〜3ヶ月 | 90記事 | 500〜8,000 | 100〜2,000円 |
| 4〜6ヶ月 | 180記事 | 8,000〜15,000 | 2,000〜4,000円 |
| 7〜12ヶ月 | 270〜360記事 | 15,000〜50,000 | 4,000〜12,500円 |
| 13〜18ヶ月 | 360〜540記事 | 50,000〜120,000 | 12,500〜30,000円 |

**現実的な月3万円達成時期: 12〜18ヶ月後**

### 重要な提言: AdSense単体に依存しない

AdSense単体では時間がかかるため、以下の併用を推奨：

| 収益源 | 施策例 | 達成短縮効果 |
|--------|--------|-------------|
| AdSense | 広告収入（メイン） | - |
| アフィリエイト | Claude Pro紹介、技術書（Amazon） | 大 |
| 有料コンテンツ | Zenn/noteで深掘り記事販売 | 中 |
| SNS連携 | X(Twitter)での発信→流入 | 大（初期は生命線） |

**複合型で6〜12ヶ月に短縮可能**

## 4. 自動投稿システム設計

### アーキテクチャ

```
┌──────────────────────────────────────────────────────┐
│              ローカル環境 (MacBook Air M4)              │
│                                                      │
│  .env          generate.py ──→ Claude API            │
│  prompts/          │                                 │
│                    ▼                                 │
│               drafts/ (下書きJSON)                     │
│                    │                                 │
│                    ▼                                 │
│               review.py (人間が確認・編集)              │
│                    │                                 │
│                    ▼                                 │
│               publish.py ──→ WordPress REST API      │
│                    │              │                   │
│               logs/              ▼                   │
│                           WordPress下書き保存          │
│                              │                       │
│                              ▼                       │
│                    管理画面で人間が「公開」ボタン         │
└──────────────────────────────────────────────────────┘
```

### 必要ライブラリ（全て軽量）

| ライブラリ | 用途 |
|-----------|------|
| `anthropic` | Claude API呼び出し |
| `requests` | WordPress REST API通信 |
| `python-dotenv` | .env読み込み |
| `python-slugify` | スラッグ生成 |

### メモリ使用量: 約30〜55MB（4GBの1.4%、全く問題なし）

### ディレクトリ構造

```
wp-auto-poster/
├── .env                  # 認証情報（Git管理外）
├── .env.example          # テンプレート
├── .gitignore
├── requirements.txt
├── generate.py           # 記事生成（Claude API）
├── review.py             # 確認・編集（ターミナルUI）
├── publish.py            # WordPress下書き投稿
├── list_drafts.py        # 下書き一覧
├── lib/                  # 共通モジュール
│   ├── config.py
│   ├── claude_client.py
│   ├── wp_client.py
│   └── draft_store.py
├── prompts/              # プロンプトテンプレート
│   ├── blog_post.txt
│   └── seo_meta.txt
├── drafts/               # 生成済み下書き（Git管理外）
└── logs/                 # 投稿ログ（Git管理外）
```

### 日常のワークフロー（3ステップ）

```bash
# ステップ1: 記事生成（1〜2分）
python generate.py --topic "Claude Codeとは？できることを完全解説"

# ステップ2: 確認・編集（5〜15分）
python review.py 2026-02-21_claude-code-intro
# → ターミナルで内容確認 → 承認 or エディタで修正

# ステップ3: WordPress下書き投稿（数秒）
python publish.py 2026-02-21_claude-code-intro
# → 管理画面URLが表示される → ブラウザで最終確認して「公開」
```

### WordPress側の事前設定

1. 「編集者」権限の専用ユーザーを作成
2. そのユーザーでアプリケーションパスワードを発行
3. REST APIの動作を `curl` で確認
4. .envに認証情報を設定

## 5. SEOキーワード戦略

### 3段階のキーワード攻略

| 段階 | 時期 | キーワード例 | 狙い |
|------|------|-------------|------|
| Tier 1（即座） | 1〜3ヶ月 | 「Claude Code MCP設定」「Claude Codeエラー解決」 | ロングテール・競合少 |
| Tier 2（中期） | 4〜6ヶ月 | 「Claude Code Cursor比較」「AIコーディング比較」 | ミドルKW |
| Tier 3（後期） | 7ヶ月〜 | 「Claude Code 使い方」「AI プログラミング」 | ビッグKW |

### 記事構成の3層構造

- **集客記事**（入門・比較系）: PVを稼ぐ → AdSenseインプレッション
- **ミドル記事**（テクニック・活用系）: 滞在時間長い → 広告クリック率高い
- **専門記事**（Cowork・設定系）: ニッチ・競合少 → コアファン獲得

## 7. セキュリティ注意事項

- `.env`は**絶対にGit管理しない**（.gitignoreに必ず記載）
- ファイルパーミッション: `chmod 600 .env`
- 管理者ではなく「編集者」権限の専用ユーザーを使用
- Claude API キーの利用上限（spending limit）を設定
- アプリケーションパスワードは3〜6ヶ月ごとにローテーション
- publish.pyは**常にdraftステータス固定**（公開は必ず手動）
