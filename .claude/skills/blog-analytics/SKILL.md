# Blog Analytics スキル

Google Analytics 4とGoogle Search Consoleのデータをブラウザ操作で取得し、ブログ運営の現状分析と改善提案を日本語でレポートする。

## 基本情報

- **デフォルト分析期間**: 過去28日間
- **対象ツール**: GA4 + Google Search Console
- **出力**: チャット内レポート + Markdownファイル（`reports/blog-analytics-YYYY-MM-DD.md`）

## 実行手順

### Step 0: 準備

1. TodoWriteでタスクリストを作成して進捗管理
2. 作業ディレクトリ確認: `reports/` フォルダが存在しない場合は作成
3. 分析期間の設定:
   - デフォルト: 過去28日間
   - `$ARGUMENTS` で期間指定がある場合はそれを使用（例: `90days`, `3months`）
4. **前回レポートの読み込み**: `reports/` 内の最新の `blog-analytics-*.md` を検索して読み込む
   - 前回レポートが存在する場合: 主要KPI（ユーザー数・セッション数・クリック数・表示回数・CTR・平均順位）を抽出してメモしておく
   - 存在しない場合: 初回分析として比較なしで進める

### Step 1: ブラウザタブの準備

```
1. tabs_context_mcp で現在のタブを確認（createIfEmpty: true）
2. 必要に応じて新しいタブを作成（tabs_create_mcp）
3. 以降の操作はすべてこのタブで実行
```

### Step 2: Google Analytics 4 データ取得

#### 2-1: GA4 概要レポートにアクセス

1. `https://analytics.google.com/` に navigate
2. ページロードを待って computer(screenshot) で現在の状態を確認
3. 正しいプロパティ（ブログのプロパティ）が選択されているか確認
   - 複数プロパティがある場合: 画面上部のプロパティ名をクリックして選択
4. 左メニューから「レポート」→「概要」を開く

#### 2-2: 期間設定

1. 画面右上の日付範囲をクリック
2. 「過去28日間」を選択（または指定期間）
3. 期間が適用されたことを screenshot で確認

#### 2-3: 概要メトリクス取得

`read_page` または `get_page_text` で以下のデータを読み取る：
- **ユーザー数**（新規 + リピーター）
- **セッション数**
- **エンゲージメント率**
- **平均エンゲージメント時間**

取得できない場合は screenshot を撮って視覚的に確認する。

#### 2-4: 人気ページ取得

1. 左メニューから「エンゲージメント」→「ページとスクリーン」を開く
2. 上位10記事のページタイトルとPV数を記録
3. `read_page` でテーブルデータを読み取る
   - うまく読み取れない場合は `javascript_tool` で DOM からテーブルデータを抽出

```javascript
// テーブルデータ抽出の例
Array.from(document.querySelectorAll('table tr')).map(row =>
  Array.from(row.querySelectorAll('td, th')).map(cell => cell.textContent.trim())
)
```

#### 2-5: トラフィック流入源

1. 「獲得」→「トラフィック獲得」を開く
2. 流入元別のセッション数を記録
   - Organic Search（検索流入）
   - Direct（直接流入）
   - Social（SNS流入）
   - Referral（被リンク流入）

### Step 3: Google Search Console データ取得

#### 3-1: Search Console にアクセス

1. `https://search.google.com/search-console/` に navigate
2. ページロードを待ち、ブログのプロパティが選択されているか確認
3. 必要に応じてプロパティを選択

#### 3-2: 検索パフォーマンス取得

1. 左メニューから「検索パフォーマンス」→「検索結果」を開く
2. 期間を過去28日間に設定
3. 「クリック数」「表示回数」「CTR」「掲載順位」のすべてにチェックを入れる

#### 3-3: 主要KPI記録

ページ上部のサマリーカードから：
- **合計クリック数**
- **合計表示回数**
- **平均CTR**
- **平均掲載順位**

#### 3-4: 検索キーワード取得（上位20件）

1. 「クエリ」タブが選択されていることを確認
2. `read_page` でテーブルを読み取り、以下を記録：
   - キーワード名
   - クリック数
   - 表示回数
   - CTR
   - 掲載順位
3. 特に「掲載順位4〜20位」のキーワードに注目（改善余地が大きい）

#### 3-5: 上位ページ取得

1. 「ページ」タブに切り替え
2. 上位10ページのURL・クリック数・表示回数を記録

#### 3-6: デバイス別データ（オプション）

「デバイス」タブでモバイル/デスクトップの比率を確認。

### Step 4: データ分析と改善提案

収集したデータを総合的に分析し、以下の観点で考察する：

#### 4-1: 前回比較評価（前回レポートがある場合）

前回レポートから抽出した数値と今回の数値を比較し、各指標の変化を算出する：

| 指標 | 前回 | 今回 | 変化 | 評価 |
|------|------|------|------|------|
| ユーザー数 | | | +X% | ↑/↓/→ |
| セッション数 | | | | |
| エンゲージメント率 | | | | |
| クリック数 | | | | |
| 表示回数 | | | | |
| 平均CTR | | | | |
| 平均掲載順位 | | | | |

評価のポイント：
- **大きく改善した指標**: なぜ伸びたか（投稿数増加・SEO施策の効果など）を考察
- **悪化した指標**: 原因の仮説を立てる
- **前回の改善提案の効果検証**: 前回レポートの「優先度High」の提案が実行されたか、効果があったか確認

#### 4-2: トラフィック状況の評価

- 流入源の健全性（検索流入が主軸かどうか）
- 人気コンテンツの傾向（テーマ・カテゴリ）

#### 4-3: SEO機会の特定

検索キーワードデータから：
- **Quick Win**: 掲載順位6〜15位で表示回数の多いキーワード → コンテンツ強化で上位狙いが可能
- **CTR改善**: 高順位だがCTRが低いキーワード → タイトル・メタ説明の改善
- **コンテンツギャップ**: 検索されているが対応記事がないテーマ

#### 4-3: 改善提案（優先度付き）

**優先度High（すぐに取り組むべき）:**
- 具体的なアクション + 期待効果

**優先度Medium（今月中に）:**
- 具体的なアクション + 期待効果

**優先度Low（来月以降）:**
- 具体的なアクション + 期待効果

## トラブルシューティング

### ログインが必要な場合
「Googleにログインしてください」とユーザーに案内し、ログイン完了後に操作を再開する。

### データが読み取れない場合
1. まず screenshot を撮って現在の状態を確認
2. `read_page` が失敗する場合は `javascript_tool` でDOMから直接抽出を試みる
3. それも失敗する場合は screenshot から視覚的に情報を読み取り記録する

### プロパティが複数ある場合
画面上部のプロパティ名をクリックして一覧を表示し、ブログに対応するプロパティをユーザーに確認する。

## 注意事項

- ブラウザ操作は時間がかかる場合がある。各ステップで適宜 screenshot で状態確認する
- データが取得できた時点でその都度メモとして記録しておく（途中でエラーが起きてもデータを失わないため）
- GA4とSearch Consoleで異なるURLを示す場合がある（短縮URLなど）。URLより記事タイトルを優先して紐付ける
- プライバシーに配慮し、個人情報を含むデータはレポートに含めない
