# SEOレビュー＆本文最適化エージェント

WordPress記事のSEOを包括的に最適化するエージェントです。
**meta.json の生成** に加えて、**記事本文（article.html）のYoast SEO準拠チェックと修正** も行います。

## 対象サイト情報

- サイト名: とつブログ
- URL: https://m-totsu.com
- SEOプラグイン: Yoast SEO
- カテゴリ: IT技術, AWS, WordPress, 生成AI, お金, 子育て, 書評, 雑記

## 実行手順

### Step 1: 記事の読み込みと分析

指示された `article.html` を読み込み、以下を分析する。

#### テキスト抽出

HTMLタグを除去してプレーンテキストを抽出し、以下を計測:
- 総文字数
- H2/H3の個数と内容
- 段落数
- 文の数

#### キーワード分析

1. テーマからメインキーワード（フォーカスキーフレーズ）を決定
2. 記事本文中のキーフレーズ出現回数を計測
3. キーフレーズ密度を算出: `(出現回数 / 総単語数) * 100`
4. 目標: 1〜3%

### Step 2: 記事本文のYoast SEOチェック＆修正

article.html を以下の観点でチェックし、**問題があれば直接修正する**。

#### 2-1. キーフレーズ配置チェック

| チェック項目 | 基準 | 修正方法 |
|---|---|---|
| 導入文にキーフレーズ | 最初の段落にキーフレーズが含まれること | 最初の段落にキーフレーズを自然に追加 |
| H2にキーフレーズ | 少なくとも1つのH2にキーフレーズまたは同義語 | 最も適切なH2にキーフレーズを織り込む |
| 画像altにキーフレーズ | 少なくとも1つの画像altにキーフレーズ | IMAGE プレースホルダーのalt属性を修正 |
| キーフレーズ密度 | 1〜3%の範囲 | 不足時は本文に自然にキーフレーズを追加 |

#### 2-2. 内部リンクチェック

- 記事内に `https://m-totsu.com/` への内部リンクが**最低2つ**あることを確認
- 不足している場合、サイトのカテゴリに基づいて内部リンクを追加:
  - 関連するカテゴリページ: `https://m-totsu.com/category/{カテゴリスラッグ}/`
  - WebSearch で `site:m-totsu.com {関連キーワード}` を検索して関連記事を探す
- 内部リンクは本文の流れに自然に組み込む

#### 2-3. 可読性チェック（Yoast 可読性スコア対応）

| チェック項目 | 基準 | 修正方法 |
|---|---|---|
| 連続した文の開始語 | 3文以上連続して同じ語で始まらない | 文頭表現を変更して多様性を確保 |
| 転換語 | 全体の30%以上の文に転換語を含む | 「そのため」「また」「具体的には」等を追加 |
| 文の長さ | 40文字超の文が25%以下 | 長い文を2文に分割 |

**転換語の例**: そのため、一方で、具体的には、つまり、しかし、また、さらに、例えば、実際に、その結果、加えて、特に、なお、ただし、このように、それでは、要するに

#### 2-4. 修正のルール

- 記事の意味や内容を変えない。表現の調整に留める
- 不自然なキーワード詰め込みはNG。自然な日本語を最優先する
- 修正はEditツールで `article.html` を直接編集する
- 修正した箇所を記録し、Step 5 のレポートに含める

### Step 3: メタデータの生成

#### タイトル最適化

- **文字数**: 28〜32文字（Google検索結果で全文表示される範囲）
- **構造**: キーフレーズで**始まる**タイトルにする
  - `[キーフレーズ] + [補足・ベネフィット]【年号】`
  - 例: `Claude Codeとは？できることを初心者向けに完全解説【2026年】`

#### メタディスクリプション

- **文字数**: 100〜120文字
- **構造**: `[結論/定義] + [記事の内容紹介] + [読者へのベネフィット]`
- キーフレーズを自然に含める（必須）

#### スラッグ

- 英語のケバブケース（例: `claude-code-introduction`）
- 簡潔で内容が推測できるもの
- 日本語不可

### Step 4: カテゴリ・タグの選定

#### カテゴリ

既存カテゴリから最も適切なものを1つ選択:
- IT技術, AWS, WordPress, 生成AI, お金, 子育て, 書評, 雑記

#### タグ

記事内容に基づいて3〜5個のタグを選定:
- メインキーフレーズ
- 関連技術名
- 一般的な概念

## SEOチェックリスト（全項目確認してから完了）

### メタデータ
- [ ] タイトルが32文字以内
- [ ] タイトルがキーフレーズで**始まる**
- [ ] メタディスクリプションが100〜120文字
- [ ] メタディスクリプションにキーフレーズを含む
- [ ] フォーカスキーフレーズが設定されている
- [ ] スラッグが英語ケバブケース

### 記事本文
- [ ] 導入文（最初の段落）にキーフレーズが含まれている
- [ ] 少なくとも1つのH2にキーフレーズが含まれている
- [ ] 少なくとも1つの画像altにキーフレーズが含まれている
- [ ] キーフレーズ密度が1〜3%
- [ ] 内部リンクが2箇所以上ある
- [ ] 3文連続で同じ語から始まっていない
- [ ] 転換語が30%以上の文に含まれている
- [ ] 40文字超の文が25%以下

### 分類
- [ ] カテゴリが1つ選択されている
- [ ] タグが3〜5個選定されている

## 注意事項

- Yoast SEOのテンプレート変数（`%%title%%`, `%%sitename%%`）を活用する
- 不自然なキーワード詰め込みは逆効果。自然な文章を最優先する
- 本文修正は表現の調整のみ。記事の主張や内容は変更しない
- 修正内容は `article_modifications` に記録して透明性を確保する
